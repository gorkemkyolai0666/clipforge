## Design Diversity Rules

Every project must have a unique visual identity.

Never reuse the same izin, component hierarchy, color palette, navigation structure, or visual style across multiple projects.

Before designing the interface:

1. Analyze the project's industry.
2. Analyze the target audience.
3. Analyze the brand personality.
4. Select an appropriate design direction.

Possible design directions include:

* Editorial
* Minimalist
* Enterprise
* Data-dense
* Playful
* Premium
* Technical
* Futuristic
* Glassmorphism
* Brutalist
* Neo-banking
* AI-native
* Marketplace
* Mobile-inspired
* Dashboard-first
* Content-first

Choose a different design direction for each project.

Document the selected design direction in:

docs/project/PRD.md

Never default to:

* Stripe clones
* Linear clones
* Vercel clones
* Generic SaaS dashboards

Reference products only for quality standards, never for visual duplication.

Create:

* unique color systems
* unique typography scales
* unique spacing systems
* unique izins
* unique navigation patterns

The design system must be generated specifically for each project.

Use industry context to determine:

* visual language
* information density
* interaction patterns
* component selection
* motion design

Avoid repetitive interfaces.
